## Architecture Summary

### Entry Point
- **`VSCodeiPadOSApp.swift`** (line 4): `@main` SwiftUI App. Creates `EditorCore` as the central `@StateObject`, wraps `ContentView` in a `KeyCommandBridge` for keyboard shortcut handling. Defines 8 VS Code-style menu command groups (File, Edit, Selection, View, Go, Run, Terminal, Help) all communicating via `NotificationCenter`.

### App Lifecycle
- **`AppDelegate.swift`**: UIKit app delegate for Stage Manager optimization (iPadOS 16+), window title management, and scene session cleanup.
- **`SceneDelegate.swift`**: Multi-window support via `UIWindowSceneDelegate`. Handles `NSUserActivity` for opening files/workspaces in new windows. Includes `WindowActivity` constants and `WindowStateManager` (in-memory window state tracking).

### Main UI – ContentView.swift (1245 lines)
Central view composing the entire IDE layout:

**Layout structure** (line 120-147 `mainLayout`):
- `IDEActivityBar` (vertical icon strip: files, search, git, debug, settings, terminal)
- Sidebar (switchable: Explorer/Search/Git/Debug) with resizable width
- Editor area: `IDETabBar` + `IDEEditorView` or `IDEWelcomeView`
- `StatusBarView` at bottom
- Collapsible `PanelView` (terminal/output) at bottom

**Overlay system** (line 149-191 `overlayViews`):
- Command Palette (Cmd+Shift+P)
- Quick Open (Cmd+P)
- Go To Symbol (Cmd+Shift+O)
- Go To Line (Ctrl+G)
- AI Assistant panel
- Workspace Trust Dialog

**VS Code Tunnel mode** (line 107-108): When `TunnelManager.isConnected`, the entire UI is replaced with `VSCodeTunnelView` (a WebView to remote VS Code).
