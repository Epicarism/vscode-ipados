## P0 Bug Fix: Tab Crash on Autocomplete

**File:** `VSCodeiPadOS/VSCodeiPadOS/Views/Editor/SyntaxHighlightingTextView.swift`

**Problem:** The `handleTab()` method at line 1357 was crashing because it modified `@Binding` values during SwiftUI's view update cycle. The original fix used `DispatchQueue.main.async`, but this is insufficient because `async` can still execute within the same runloop iteration when SwiftUI is actively updating views.

**Root Cause:** When pressing Tab during autocomplete, `onAcceptAutocomplete?()` is called which modifies SwiftUI `@Binding` properties (like `text: Binding<String>` at line 13). If this happens during a view update cycle, SwiftUI crashes because bindings cannot be modified while a view is being rendered.

**Fix Applied (lines 1358-1366):**
- Changed from `DispatchQueue.main.async` to `DispatchQueue.main.asyncAfter(deadline: .now())`
- `asyncAfter(.now())` guarantees execution in a **future** runloop iteration, after SwiftUI's current update cycle completes
- This is the standard iOS pattern for deferring state modifications that conflict with view updates

**After:**
```swift

```
