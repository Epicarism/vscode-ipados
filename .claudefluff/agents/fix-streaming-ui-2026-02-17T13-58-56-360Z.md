- **When `aiManager.streamingResponse` is not empty**: Displays a full assistant message bubble (matching `ChatMessageView` styling) with:

- **When `aiManager.streamingResponse` is empty** (tokens haven't arrived yet): Falls back to the original "Thinking..." spinner with `ProgressView` and caption text, tagged with `.id("loading")`

**2. Added auto-scroll on streaming token updates (lines 119-130)**

Added a new `.onChange(of: aiManager.streamingResponse)` modifier on the `ScrollView` that:
- Only fires when `aiManager.isLoading` is true
- Uses a smooth `easeOut(duration: 0.15)` animation

- This ensures the view stays scrolled to the bottom as new tokens appear in real-time

### How It Works
The `AIManager` class (verified via search) has `@Published var streamingResponse = ""` that gets cleared at the start of each request and progressively appended to as SSE/streaming tokens arrive. Since `AIAssistantView` uses `@StateObject private var aiManager = AIManager()`, SwiftUI automatically re-renders whenever `streamingResponse` changes, showing tokens in real-time.
