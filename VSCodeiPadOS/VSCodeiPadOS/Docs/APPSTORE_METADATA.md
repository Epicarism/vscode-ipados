### Export Compliance Justification

CodePad Pro qualifies for the ENC Unrestricted exemption (formerly Section 742.15(b)) because:

1. **HTTPS/TLS** — All API calls to AI providers (OpenAI, Anthropic, Google, etc.) use standard HTTPS/TLS 1.2+, which is an industry-standard protocol implemented by Apple's URLSession framework. No custom cryptography is involved.

2. **SSH** — Terminal connections and git remote operations use SSH, an industry-standard protocol implemented via SwiftNIO SSH. This is standard public-key cryptography used for user-initiated connections to servers the user explicitly configures.

3. **No custom encryption** — The app does not implement any proprietary encryption algorithms, key management systems, or cryptographic protocols beyond what is provided by Apple's platform frameworks.

4. **No encryption as primary feature** — Encryption is not a selling point or primary feature of the app; it is incidental to secure communication with user-configured endpoints.

5. **No government/military use** — The app is not designed for government, military, or restricted-entity use.

## 10. App Store Review Notes

> **Information for Apple Review Team:**

CodePad Pro is a native iPad code editor built with SwiftUI. Key points for review:

1. **No arbitrary code execution** — The app does NOT download or execute arbitrary code from the internet. All code editing is performed locally on-device. The editor displays and manipulates text; it does not compile or run user code locally.

2. **SSH terminal** — Connects only to user-configured remote servers. The user must manually provide host address, port, username, and credentials. The app does not maintain, suggest, or advertise any server list. SSH is used solely for remote shell access.

3. **AI features** — Use user-provided API keys to connect to standard AI service providers (OpenAI, Anthropic, Google, DeepSeek, Mistral, Ollama). No API keys are bundled with the app. On-device MLX models run entirely locally with no network communication whatsoever.

4. **VS Code Connected Mode** — Loads a WebView to user-configured server URLs only (code-server, VS Code Tunnel, Codespaces). The user must explicitly provide the URL. No default or pre-configured servers are provided.

5. **Git operations** — Performed locally on-device or via user-configured SSH connections. No third-party git hosting service accounts are required. Works with GitHub, GitLab, Bitbucket, or any git host.

6. **No tracking** — No App Tracking Transparency popup is needed. The app performs no tracking or analytics of any kind. There are no third-party SDKs that collect data.

7. **Credential storage** — Uses standard iOS Keychain for secure credential storage. No credentials are transmitted to any server other than the user-configured endpoints.

8. **Open source** — The full source code is available at https://github.com/nicktmro/vscode-ipados for full transparency.

**To test the app:**
1. Open CodePad Pro → tap **"New File"** → create and edit a file
2. Use the file tree sidebar to browse, create, and organize files
3. Open the **Command Palette** with `Cmd+Shift+P` — browse available commands
4. Open **Quick Open** with `Cmd+P` — search for and open files
5. Test syntax highlighting by opening any supported file type (`.swift`, `.py`, `.js`, `.go`, `.rs`, etc.)
6. Switch themes: **Cmd+Shift+P** → type "Theme" → select any of the 21 themes
7. To test AI features: **Settings → AI** → enter a valid API key for any supported provider → use inline suggestions or the AI chat panel
8. Terminal features: **Settings → Terminal** → configure an SSH connection → connect to a server
9. Git features: Open a folder → **Source Control** → **Init** to create a repo, or **Clone** via SSH URL
10. Try split editor: open a file → **Cmd+Shift+P** → "Split Editor Right"

**No demo accounts needed.** All core editing features work with local files by default. AI and SSH features require user configuration.
