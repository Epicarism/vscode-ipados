## 10. App Store Review Notes

> **Information for Apple Review Team:**

CodePad Pro is a native iPad code editor. Key points for review:

1. **No arbitrary code execution** — The app does NOT download or execute arbitrary code from the internet. All code editing is performed locally on-device.
2. **SSH terminal** — Connects only to user-configured remote servers. The user must manually provide host address and credentials. The app does not maintain or advertise any server list.
3. **AI features** — Use user-provided API keys to connect to standard AI service providers (OpenAI, Anthropic, Google). No API keys are bundled with the app. On-device MLX models run entirely locally with no network communication.
4. **VS Code Connected Mode** — Loads a WebView to user-configured server URLs only (code-server, VS Code Tunnel, Codespaces). The user must provide the URL.
5. **Git operations** — Performed locally on-device or via user-configured SSH connections. No third-party git hosting service accounts are required.
6. **No tracking** — No App Tracking Transparency required. The app performs no tracking or analytics of any kind.
7. **Credential storage** — Uses standard iOS Keychain for secure credential storage. No credentials are transmitted to any server other than the user-configured endpoints.
8. **Open source** — The full source code is available at https://github.com/nicktmro/vscode-ipados

**To test the app:**
1. Open CodePad Pro and tap "New File" to create a file
2. Use the file tree sidebar to browse and create files
3. Open the Command Palette with Cmd+Shift+P
4. Test syntax highlighting by opening any supported file type (.swift, .py, .js, etc.)
5. To test AI features, go to Settings > AI and enter a valid API key for any supported provider
6. Terminal features require configuring an SSH connection in Settings > Terminal
7. Git features work with local repositories — use Init to create a new repo or Clone via SSH

No demo accounts needed. All core editing features work with local files by default.
